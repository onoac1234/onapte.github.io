Beginning with React

So, after having worked with django and getting hang of some really good backend practices, I have now decided to dive into the frontend part. I feel that attempting to create some rich user interfaces with just html, CSS and Vanill Js is not feasible. For this purpose, I have decided to start with React. Ofcourse, I could have choosen other frontend frameworks, but again, React wins as it is currently the most popular.

There are a couple of more reasons why I feel one must know frontend frameworks.

Vanilla Js is too complicated for large apps

Look at the below code. 

var myNodelist = document.getElementsByTagName("LI");
var i;
for (i = 0; i < myNodelist.length; i++) {
    var span = document.createElement("SPAN");
    var txt = document.createTextNode("\u00D7");
    span.className = "close";
    span.appendChild(txt);
    myNodelist[i].appendChild(span);
}

// Click on a close button to hide the current list item
var close = document.getElementsByClassName("close");
var i;
for (i = 0; i < close.length; i++) {
    close[i].onclick = function() {
    var div = this.parentElement;
    div.style.display = "none";
  }
}

// Add a "checked" symbol when clicking on a list item
var list = document.querySelector('ul');
list.addEventListener('click', function(ev) {
    if (ev.target.tagName === 'LI') {
        ev.target.classList.toggle('checked');
    }
    }, false);

Just to create a simple todo list in Vanilla Js takes a lot of time (and effort).

No code reusability in Vanilla js

This one's pretty obvious! It is one of the reasons why Vanilla Js is not implemented for large apps.

Ofcourse, my aim is not to degrade Vanilla Js, but it is pretty clear that it is clearly not the best to use for particularly larger apps. So, starting from the next blog I will continue to update what concepts I have learned or what projects I have done or being doing in React.
Stay tuned and let the new journey begin!
